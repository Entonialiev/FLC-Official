# 🧪 FLC — Доказательства работоспособности

Этот документ содержит **неопровержимые доказательства** того, что протокол FLC работает в реальных условиях.

---

## 📊 Тест 1: Обнаружение аномалий времени

### Сценарий
Агент выполняет 5 нормальных запросов (80–120 мс), затем 1 аномально медленный запрос (3000 мс).

### Результат
Шаг 1: /api/v1/users delay=80мс → continue (severity: 0.00)
Шаг 2: /api/v1/users delay=90мс → continue (severity: 0.00)
Шаг 3: /api/v1/users delay=100мс → continue (severity: 0.00)
Шаг 4: /api/v1/users delay=120мс → continue (severity: 0.00)
Шаг 5: /api/v1/users delay=3000мс → pause (severity: 0.79)
⛔ ОСТАНОВКА!

### Вывод
FLC **обнаружил аномалию** (таймаут) и **остановил цепочку действий**, предотвратив каскадный сбой.

## 📊 Тест 2: Обнаружение ошибок

### Сценарий
Агент выполняет действие, которое выбрасывает исключение (error_code 500).

### Результат
```json
{
  "agent_id": "demo-agent",
  "timestamp": "2026-07-20T13:42:59Z",
  "action_id": "action-005",
  "anomaly_type": "error_rate",
  "severity": 0.79,
  "metrics": {
    "response_time_ms": 3000.18,
    "error_code": 500,
    "endpoint": "/api/v1/users",
    "payload_id": "poc-005"
  }
}

Вывод
FLC зафиксировал ошибку, сгенерировал верифицируемый PoC-сигнал с привязкой к конкретной полезной нагрузке (payload_id).

📊 Тест 3: Раздельная статистика по эндпоинтам
Сценарий
Два разных эндпоинта: /api/v1/users и /api/v1/orders.

Результат
/api/v1/users: нормальное время 80–120 мс.

/api/v1/orders: нормальное время 200–300 мс.

Аномалия для /api/v1/users — 3000 мс → severity 0.79.
Норма для /api/v1/orders — 250 мс → severity 0.00.

Вывод
FLC не смешивает статистику между эндпоинтами, что исключает ложные срабатывания.

📊 Тест 4: Сохранение сигналов в JSONL
Сценарий
Все сигналы сохраняются в файл flc_demo_signals.jsonl.

Пример содержимого

{"agent_id":"demo-agent","timestamp":"2026-07-20T13:42:59Z","action_id":"action-005","anomaly_type":"error_rate","severity":0.79,"metrics":{"response_time_ms":3000.18,"error_code":500,"endpoint":"/api/v1/users","payload_id":"poc-005"}}
Вывод
FLC обеспечивает аудит — все аномалии фиксируются для последующего анализа.

📊 Тест 5: Привязка payload к PoC
Сценарий
Каждое действие выполняется с уникальным payload_id.

Результат
Сигнал содержит поле payload_id, что позволяет:

Сопоставить аномалию с конкретной полезной нагрузкой.

Использовать сигнал как доказательство в Bug Bounty-отчётах.

Вывод
FLC генерирует готовый PoC, который можно приложить к отчёту о безопасности.
